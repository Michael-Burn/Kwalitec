# Roadmap — Version 3

**Status:** Planning reference (post–Version 2 close)  
**Date:** 2026-07-20  
**Predecessor:** Version 2.0.0 (`v2.0.0`) — Educational Operating System closed by APP-004  

---

## Purpose

Version 2 delivered the Educational Operating System: deterministic educational cores, composition root, optional AI enrichment boundary, architecture governance, and production operational readiness.

Version 3 must **not** reopen Version 2 architecture. It advances product evidence, cutover confidence, and measured educational outcomes.

---

## Guiding Constraints

1. **Evidence before features** — Internal Alpha behaviour, Founder operational health, repeated feedback themes, and learning-science research justify scope (see Version 2 product strategy).
2. **Constitution remains binding** — Educational truth, determinism, explainability, AI enrichment boundary, and dependency inversion stay enforced.
3. **No silent cutover** — Version 1 retirement (V2-020) remains an explicit, gated programme — not a side effect of Version 3 work.
4. **Operational bar** — Production readiness patterns from APP-004 (typed config, observability, resilience, CI gates) apply to all new runtime surfaces.

---

## Candidate Themes (not a backlog)

| Theme | Intent | Must not |
|---|---|---|
| Evidence-gated product depth | Strengthen Student / Session experience where Alpha evidence shows friction | Invent educational math in UI |
| Sole-runtime cutover | Complete dual-run evidence gates and Version 1 retirement when ready | Flip traffic without explainability and continuity proof |
| Curriculum operations | Harden Studio / ingestion / publication for multi-subject ops | Bypass Curriculum Management policies |
| Founder advisory insight | Extend non-authoritative operational signals | Let Founder tools mutate learner journeys as a side channel |
| Enrichment quality (optional) | Improve presentation enrichment providers behind the same port | Move AI into domain decision paths |

---

## Entry Criteria for Version 3 Work

- Version 2 tagged `v2.0.0` with release artefacts present
- Architecture governance CI green
- APP-004 operational checklist satisfied for the EOS runtime
- Any educational change accompanied by an ADR or explicit constitution check when boundaries are touched

---

## Exit Criteria (Version 3 — when defined)

Version 3 will close only when a separate programme defines measurable educational and operational outcomes. This document does not invent those outcomes in advance.
