# Private Beta — Issue Reporting

**Status:** Process prepared

## Goal

Reliable defect intake without exposing other students’ data or production secrets.

## Reporter guide

1. Steps to reproduce
2. Expected vs actual
3. URL / surface (`/student/...`, `/session/...`)
4. Timestamp and account email (beta only)
5. Screenshots without secrets
6. Browser / device

## Operator guide

- Classify: security / educational integrity / functional / UI / performance
- Security issues: private channel; do not discuss exploits in public issues
- Educational integrity: freeze speculative “fixes” to algorithms; STOP → Document → Recommend
- Link to Technical Debt if deferred intentionally

## Tracking

Use the repository issue tracker or founder ops board. Tag `private-beta`.
