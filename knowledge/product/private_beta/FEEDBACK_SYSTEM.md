# Private Beta — Feedback System

**Status:** Process prepared

## Goal

Capture feedback that improves learning outcomes and trust — not vanity feature requests.

## Channels

| Channel | Use |
|---|---|
| In-product founder feedback (existing) | Structured product feedback |
| Scheduled check-ins | Consistency, confusion, trust |
| Vision Final Test framing | “Did this help you study like a professional?” |

## Capture fields (minimum)

- What were you trying to do?
- What did you expect?
- What happened?
- Did you understand **why** the recommendation was made?
- Severity: blocker / confusing / suggestion
- Consent to quote anonymously

## Triage

1. Educational honesty / wrong next action → High (educational governance)
2. Lost / overwhelmed / unclear navigation → High (experience principles)
3. Polish / preference → Medium/Low
4. Activity-metric feature requests that violate Never Build → Decline with Vision citation

## Storage

Prefer existing founder feedback and research folders; do not stand up a new SaaS without privacy review.
