# Private Beta — Support Workflow

**Status:** Process prepared

## Goal

Students feel guided and supported (Vision experience principles) with a clear human path when stuck.

## Support tiers

| Tier | Response target (beta) | Examples |
|---|---|---|
| P0 Security / data | Immediate | Account takeover, data leak |
| P1 Cannot study | Same day | Login failure, session broken, wrong student data |
| P2 Confusing guidance | 1–2 business days | Unclear recommendation, navigation confusion |
| P3 Suggestion | Weekly batch | UX ideas, copy nits |

## Workflow

1. Intake (feedback or issue report)
2. Triage (Issue Reporting)
3. Reproduce on staging / dual-run local if needed
4. Fix via normal PR standards **or** document + recommend if educational algorithm involved
5. Confirm with reporter
6. Note in release notes if user-visible

## Ownership

Founder / designated beta operator owns triage until a support role exists.
