## 1. Programme Vision

The Educational Integrity Programme exists so that every educational behaviour in Kwalitec is consistent with the Educational Constitution and capable of passing the Educational Governance Review Standard.

Kwalitec’s commercial and educational obligation is student trust. Trust requires more than polished surfaces and working workflows. Students must receive one coherent educational story: what they have covered, what the system estimates about understanding, what they should do today, why that advice exists, and what remains their choice.

Educational Philosophy Audit Version 2 established that Version 1.0 educational readiness fails not primarily because of scattered defects, but because a small set of governance violations still allow unlawful educational inference, mixed state ownership, and conflicting student-facing narration.

This Programme therefore consolidates remaining educational work into five sequential capabilities. Each capability closes one class of educational problem. No capability invents new educational philosophy. No capability redesigns Educational Intelligence or the Digital Twin. Implementation proceeds only under constitutional meaning already declared by EGI-001 through EGI-003.

**End state:** Kwalitec’s educational cores are constitutionally lawful, evidence-bound, narratively coherent, and governance-approved for Version 1.0 educational release.

## 2. Educational Problem Statement

Educational Philosophy Audit Version 2 scored current product behaviour at **28 / 100** Educational Governance Score — Failed governance, formal outcome **REJECTED** under EGI-003. Categories A (Constitution), B (Logic Registry), and D (Evidence Integrity) were NON-COMPLIANT. Educational Alignment was weak. Version 1.0 Educational Release Gate and Internal Alpha educational deployment both fail.

The Audit also recorded known strengths: Learning Mode mission authority is constitutionally correct; Study Progress coverage badges no longer invent Mastered theatre; Digital Twin naming is largely invisible; curriculum-first deterministic cores remain free of opaque generative mission selection. Those strengths prove that governed educational behaviour is achievable. They do not clear the understanding and evidence spine.

The underlying educational failures are not isolated bugs. They reduce to a small number of root causes:

### Root Cause 1 — Mixed educational state ownership

Distinct educational questions share writers, fields, or labels. Study Progress, Estimated Mastery, student-felt confidence, and readiness composites can author or imply one another. When one state writes another, student-facing truth collapses.

### Root Cause 2 — Unlawful evidence succession

Completion events, elapsed time, and self-reported confidence can still influence Estimated Knowledge or Estimated Mastery without constitutionally sufficient Educational Evidence. Soft signals masquerade as understanding.

### Root Cause 3 — Fragmented educational narrative

Student-facing surfaces do not always tell one story. Mission authority, advisory recommendations, dashboard metrics, and analytics readiness can disagree without mandatory disclosure of educational role (Observed Fact, Derived Fact, Educational Estimate, or Educational Advice).

### Root Cause 4 — Governance verification lag

Stabilisation doctrines (including IA-001, IA-003, IA-004) improved speech and mission integrity, but Categories A–D remain open under automatic NON-COMPLIANT triggers. Engineering correctness advanced faster than educational approval.

### Root Cause 5 — Version 1 readiness is undefined by governance closure

Version 1.0 educational maturity cannot be declared by feature count. It must be declared by Governance Review outcome, Constitution/Registry compliance, Internal Alpha educational trust confirmation, and intentional documentation of residual debt.

**Implication:** Fixing findings one surface at a time would recreate fragmentation. This Programme addresses root causes in dependency order so that evidence integrity and narration rest on single ownership, and Version 1 readiness rests on verified governance — not aspiration.

## 3. Programme Objectives

The Programme succeeds only when the following objectives are met measurably.

| ID | Objective | Measurable signal |
|----|-----------|-------------------|
| O-1 | **One educational state has one owner** | Educational State Ownership Matrix intact; no conflicting writers for Study Progress, Educational Evidence, Estimated Knowledge, Estimated Mastery, Current Learning Topic, or Today’s Mission |

Supporting objectives:

- Learning Mode remains the authority for Today’s Mission persistence unless a future constitutionally authorised mode amends that rule.
- Regression protection exists for every closed educational integrity violation.
- Residual educational debt, if any, is intentional, owned, and documented — never silent.

## 4. Programme Principles

Every capability in this Programme obeys the following principles.

### P-1 — Root causes before symptoms

Capabilities close ownership, evidence, and narrative root causes. They do not ship cosmetic copy patches over unlawful inference.

### P-2 — Constitution before implementation

If proposed behaviour would contradict the Constitution, the Constitution must be amended first under Article X. Implementation never redefines educational meaning in code.

### P-3 — Logic before algorithms

Educational Logic Registry entries declare behaviour before algorithms realise it. Unregistered educational decisions that students rely on are forbidden.

### P-4 — Educational truth before optimisation

Trustworthy weaker guidance is preferred to optimised guidance that invents certainty, mixes states, or silently commandeers Learning Mode.

### P-5 — One capability solves one educational problem

Each EIP capability has one educational purpose. Cross-cutting convenience refactors that blur problem boundaries are out of scope for that capability.

### P-6 — Regression protection required

Every closed integrity violation must leave automated regression protection proportional to educational risk. Manual demonstration alone is insufficient for APPROVED outcomes.

### P-7 — No dependency skipping

Capabilities execute in dependency order. Narrative work may not declare victory while ownership and evidence remain open. Version 1 readiness may not declare victory while Governance Review remains REJECTED or conditional.

### P-8 — Scope discipline

This Programme does not redesign Educational Intelligence, the Digital Twin, Revision Mode, Diagnostic Mode, or Adaptive Learning product surfaces beyond what is necessary to restore constitutional integrity of existing educational cores.

## 5. Capability Roadmap

Implementation sequence is fixed. Later capabilities consume the integrity established by earlier ones.

---

### Capability 1 — EIP-001 Educational State Ownership

**Purpose**  
Establish single ownership for every educational state that students and educational cores rely upon.

**Educational problem addressed**  
Mixed writers and overloaded fields allowing estimates, coverage, confidence, and readiness to author one another.

**In scope (educational)**  
- Study Progress  
- Educational Evidence  
- Estimated Knowledge  
- Estimated Mastery  
- Current Learning Topic  
- Today’s Mission / Learning Mode persistence  

**Expected outcome**  
These states are clearly separated in meaning, ownership, and succession rights. Estimated Mastery never mints Study Progress. Coverage events never impersonate mastery. Confidence fields do not co-write coverage as competence. Dual or conflicting readiness meanings under one student label are resolved or lawfully renamed.

**Governing references (illustrative)**  
- Constitution: Articles III, IV, VIII (state meaning and ownership integrity)  
- Registry: EL-001, EL-002, EL-003, EL-005, EL-006, EL-007, EL-009; Educational State Ownership Matrix  
- Governance Categories: primarily A, B, E; contributes to D, G, H  

**Dependency**  
None within this Programme. Prerequisite: EGI-001–EGI-003 remain governing; EPA-002 remains the diagnostic baseline.

---

### Capability 2 — EIP-002 Evidence Integrity

**Purpose**  
Ensure only constitutionally valid evidence can influence educational estimates.

**Educational problem addressed**  
Unlawful succession from completion, self-reported confidence, thin history, or interim estimate stores into Estimated Knowledge / Estimated Mastery.

**In scope (educational)**  
- Mission Completion vs mastery recalculation boundaries  
- Student-felt confidence as Rank-D soft signal only  
- Evidence density / accumulation before high estimate language  
- Separation of coverage completion from understanding claims  

**Expected outcome**  
Completion, confidence, and elapsed time cannot directly create educational knowledge or mastery certainty. High Estimated Mastery language requires lawful Educational Evidence of sufficient quality and accumulation. Soft signals may inform advice; they do not author verified attainment.

**Governing references (illustrative)**  
- Constitution: Articles II, III, IV, V, VIII (truth, evidence, claim lawfulness)  
- Registry: EL-004, EL-005, EL-006, EL-007  
- Governance Categories: primarily A, B, C, D; contributes to H  

**Dependency**  
Requires EIP-001. Evidence rules are meaningless if states still share writers.

---

### Capability 3 — EIP-003 Educational Narrative

**Purpose**  
Ensure every educational surface tells one coherent educational story.

**Educational problem addressed**  
Fragmented student-facing narration: advisory divergence without disclosure; metric label conflict; generic Why; residual vocabulary that blurs estimate and fact.

**In scope (educational)**  
- Mission presentation  
- Reason / explainability in educational language  
- Advice classification and disclosure when advisory focus differs from Learning Mode  
- Student Choice (agency without mastery mutation)  

**Expected outcome**

```
Mission
   ↓
Reason
   ↓
Advice
   ↓
Student Choice
```

Across Mission, Dashboard, Recommendations, Study Plan, and Analytics, the student encounters one educational story with lawful claim types and clear authority roles.

**Governing references (illustrative)**  
- Constitution: Articles I, II, VI, VII, VIII (purpose, explainability, Learning Mode, messaging)  
- Registry: EL-003, EL-008, EL-009, EL-010, EL-011  
- Governance Categories: primarily C, F, G, H; requires A/B/D already restored  

**Dependency**  
Requires EIP-001 and EIP-002. Narrative coherence cannot paper over ownership or evidence violations.

**Dependency**

## 6. Dependency Graph

No capability may skip dependencies. Parallel exploratory design may occur, but implementation approval and educational completion claims follow this chain only.

```
EIP-001 Educational State Ownership
              ↓
EIP-002 Evidence Integrity
              ↓
EIP-003 Educational Narrative
              ↓
EIP-004 Educational Governance Verification
              ↓
EIP-005 Version 1 Educational Readiness
```

### Dependency rules

1. **EIP-002 may not complete** while EIP-001 leaves mixed ownership open.  
2. **EIP-003 may not complete** while EIP-002 leaves unlawful evidence succession open.  
3. **EIP-004 may not claim APPROVED** while EIP-001–EIP-003 educational purposes remain unmet.  
4. **EIP-005 may not declare Version 1 educational readiness** while EIP-004 outcome is REJECTED or REQUIRES REVISION, or while APPROVED WITH CONDITIONS remain uncleared for the Version 1.0 Educational Release Gate.  
5. Surface-level backlog items from EPA-002 must be mapped into the earliest capability that closes their root cause; they must not create a parallel unordered fix train outside this graph.

### Mapping note

EPA-002’s prioritised backlog (ownership separation, confidence decoupling, completion isolation, advisory narration, evidence density, readiness meaning, continuity, vocabulary, and related items) is absorbed into EIP-001–EIP-003 by root cause. EIP-004 and EIP-005 govern verification and readiness. Individual backlog titles may be used as delivery work packages inside a capability; they do not become independent programme tracks.

## 8. Exit Criteria

The Educational Integrity Programme is complete only when **all** of the following are true:

1. **Educational Governance Review = APPROVED** under EGI-003 (conditions cleared for Version 1.0 Educational Release Gate).  
2. **Educational Philosophy Audit** (successor or re-run of EPA-002 method) **passes** against Constitution, Registry, and Review Standard.  
3. **Internal Alpha confirms educational trust** — students can rely on educational surfaces without the week_001 trust-collapse modes remaining open.  
4. **Remaining educational debt is intentional and documented** — owned, time-bounded if temporary, and never silent divergence from Constitution or Registry.

If any exit criterion fails, the Programme remains open. Engineering release activity may proceed for non-educational scope only under Architecture Office risk acceptance; that acceptance is not Programme exit.

## 9. Governance

Every capability under this Programme must, before implementation is considered complete:

### G-1 — Cite Constitution Articles

Name the Articles and Sections that authorise the educational meaning being implemented.

### G-2 — Cite Educational Logic IDs

Name the EL-IDs from the Registry that define the behaviour. Update the Registry under EGI process **before** approving behaviour that changes registered logic.

### G-3 — Cite Governance Categories

Map work to Categories A–H and record expected rating movement.

### G-4 — Define regression tests

Specify and land automated regression protection for the educational integrity closed by the capability.

### G-5 — Complete Architecture Review

Return for Architecture Review before claiming capability completion. Educational Governance Review under EGI-003 is mandatory for educational approval; Architecture Review remains required for structural and layering impact.

### G-6 — Scope boundaries

Capabilities must not:

- modify Constitution, Registry, or Review Standard without separate governed amendment work;  
- redesign Educational Intelligence product architecture;  
- redesign Digital Twin architecture;  
- introduce black-box generative substitution into educational cores;  
- skip the dependency graph to chase isolated findings.

### G-7 — Completion artefacts

Each capability completion report must include: educational purpose, Constitution/Registry/Category citations, tests executed, migration impact, architecture compliance (including curriculum V1/V2 where applicable), residual debt, and known limitations.
